# Ollama Setup (Docker/Homebrew)

Run local AI models with Ollama. This is the simpler alternative to llama.cpp.

> **Note**: For AI coding assistants like OpenCode that require tool calling, consider using [llama.cpp](../README.md) instead. Ollama's OpenAI-compatible API (`/v1` endpoints) doesn't properly translate tool calls for most local models, causing malformed responses instead of proper JSON tool calls.

## Quick Start

### Linux (Docker + ROCm)

```bash
cd ollama
./setup.sh
```

### macOS (Homebrew)

```bash
cd ollama
./setup-macos.sh
```

> **Note**: macOS ships with Bash 3.2 which is too old for this script. If you see a version error, install a newer Bash with `brew install bash`, then run: `/opt/homebrew/bin/bash ./setup-macos.sh`

The setup scripts will:

1. Check all dependencies (and help you install missing ones)
2. Detect your GPU and configure the correct settings
3. Install and start Ollama (Docker on Linux, native on macOS)
4. Show an interactive menu to select which models to install
5. Configure OpenCode integration

## Project Structure

```text
ollama/
├── setup.sh              # Main setup script (Linux/Docker)
├── setup-macos.sh        # macOS setup script (Homebrew)
├── sync-opencode.sh      # Sync OpenCode config with installed models
├── uninstall.sh          # Clean removal script
├── docker-compose.yml    # Docker container configuration
├── models.conf           # Model selection menu configuration
├── models-metadata.conf  # Model display names and limits (auto-generated)
├── models-metadata.conf.example  # Template for metadata
├── .env.example          # Template for configuration
└── .env                  # Your local config (generated by setup)

# In parent directory:
../agent/AGENTS.md            # OpenCode system prompt template
../agent/*.md                 # Custom agent templates (plan, review, debug)
```

## Requirements

### Linux with AMD GPU

| Requirement | Notes |
|-------------|-------|
| **Linux** | ROCm (AMD's GPU compute software) only supports Linux |
| **AMD GPU** | RDNA1, RDNA2, or RDNA3 (RX 5000/6000/7000 series) |
| **Docker** | Container runtime for Ollama |
| **Disk space** | 20-100GB depending on model choices |
| **GPU Memory (VRAM)** | 8GB minimum, 16-24GB recommended |

**Dependencies** (install before running setup):

| Dependency | Install Command | Notes |
|------------|-----------------|-------|
| **Docker** | `sudo pacman -S docker` (Arch) / `sudo apt install docker.io` (Ubuntu) | Container runtime |
| **Docker Compose** | Usually included with Docker | Container orchestration |
| **curl** | `sudo pacman -S curl` (Arch) / `sudo apt install curl` (Ubuntu) | HTTP client |
| **bc** | `sudo pacman -S bc` (Arch) / `sudo apt install bc` (Ubuntu) | Calculator for size math |
| **gum** | `sudo pacman -S gum` (Arch) / See [gum install guide](https://github.com/charmbracelet/gum#installation) | Interactive menus |

### macOS Requirements

| Requirement | Notes |
|-------------|-------|
| **macOS 12+** | Monterey or later |
| **Disk space** | 20-60GB depending on model choices |
| **Apple Silicon** | Recommended - uses Metal for GPU acceleration |
| **Intel Mac** | Supported but slower (CPU only) |

## Choosing Models

The setup script shows an interactive menu where you can select which AI models to install. Models are automatically tagged based on your hardware:

- `[✓ recommended]` - Will run well on your GPU/system
- `[⚠ may struggle]` - Might work but could be slow
- `[✗ won't fit]` - Too large for your GPU memory
- `★` (star prefix) - Already installed on your system

### Model Size Guide

| Your GPU Memory | Recommended Model Sizes |
|-----------------|------------------------|
| 8GB | Up to 7B models |
| 12GB | Up to 14B models |
| 16GB | Up to 14B-20B models |
| 24GB+ | 32B+ models work great |

### Adding More Models Later

```bash
# Linux (Docker)
docker exec ollama ollama pull qwen3:14b

# macOS (native)
ollama pull qwen3:14b

# Sync to OpenCode
./sync-opencode.sh
```

## Usage

### With OpenCode

```bash
cd ~/my-project
opencode
# Inside OpenCode, type /models to switch between local models
```

### Command Line Chat

```bash
# Linux (Docker)
docker exec -it ollama ollama run qwen3:14b

# macOS (native)
ollama run qwen3:14b
```

### Managing Models

```bash
# List installed models
docker exec ollama ollama list          # Linux
ollama list                              # macOS

# Remove a model
docker exec ollama ollama rm qwen3:14b  # Linux
ollama rm qwen3:14b                      # macOS
```

### Starting and Stopping

```bash
# Linux (Docker) - run from ollama/ directory
docker compose up -d      # Start
docker compose down       # Stop
docker compose restart    # Restart

# macOS (Homebrew)
brew services start ollama
brew services stop ollama
brew services restart ollama
```

## Troubleshooting

### "Permission denied" or GPU Not Working

```bash
./setup.sh --fix-permissions
# Then log out and log back in
```

### Docker Not Running

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Model Downloads Failing

```bash
df -h ~/.ollama  # Check disk space
docker exec ollama ollama rm qwen3:14b   # Remove partial
docker exec ollama ollama pull qwen3:14b # Retry
```

### Check Status

```bash
./setup.sh --status        # Linux
./setup-macos.sh --status  # macOS
```

## Configuration

### Setup Script Options

```bash
./setup.sh --help              # Show all options
./setup.sh --status            # Check Ollama status
./setup.sh --update            # Update to latest version
./setup.sh --fix-permissions   # Fix GPU access permissions
./setup.sh --reset-agents      # Reset AGENTS.md to default
./setup.sh --skip-models       # Re-run setup without model selection
./setup.sh --force-env         # Regenerate configuration
./setup.sh --non-interactive   # Use defaults, no prompts
```

### Environment Variables

The setup script creates a `.env` file with your system's configuration:

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_NUM_CTX` | `32768` | Context window size in tokens |
| `OLLAMA_NUM_GPU` | `-1` | GPU layers (-1 = all on GPU) |
| `OLLAMA_REQUEST_TIMEOUT` | `300` | Request timeout in seconds |
| `OLLAMA_KEEP_ALIVE` | `10m` | Keep models loaded in GPU memory |

### OpenCode Configuration

To sync after adding new models:

```bash
./sync-opencode.sh              # Sync models and agents
./sync-opencode.sh --models     # Sync only models
./sync-opencode.sh --agents     # Sync only agent files
./sync-opencode.sh --merge      # Add new models, keep existing
./sync-opencode.sh --reset-agents  # Reset agents to defaults
./sync-opencode.sh --dry-run    # Preview changes
```

### Uninstalling

```bash
./uninstall.sh           # Interactive
./uninstall.sh --all     # Remove everything
./uninstall.sh --dry-run # Preview
```

## Performance Tuning

If models seem confused or forget context, increase the context window:

```bash
# Edit .env
OLLAMA_NUM_CTX=65536

# Restart
docker compose down && docker compose up -d
```
