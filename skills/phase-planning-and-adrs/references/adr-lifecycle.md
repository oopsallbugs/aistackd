# ADR Lifecycle

Use this lifecycle to keep decision records short, clear, and actionable.

## Status Flow
1. Proposed
2. Accepted
3. Superseded
4. Rejected

## Required Fields
- Context
- Decision
- Consequences
- Alternatives considered
- Links to affected code/docs/tests

## Candidate Selection Rules
- Create an ADR when a decision changes architecture boundaries.
- Create an ADR when a public contract is changed.
- Create an ADR for significant platform or operational policy choices.
- Create an ADR when major scope deferral is approved.

## Review Pattern
1. Draft decision and alternatives.
2. Review with decision owner and impacted stakeholders.
3. Accept or reject with explicit rationale.
4. Link accepted ADR to phase milestone and acceptance gate.
